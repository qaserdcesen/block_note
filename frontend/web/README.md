# Telegram Mini App (Web)

Веб-клиент теперь ориентирован на Telegram Mini App: он работает как одиночная страница, подтягивает Telegram theme/haptics, но ходит в тот же REST API (`/api/v1`), что и раньше.

## Структура
- `public/` — готовая к запуску версия мини-приложения (`index.html`, `app.js`, `styles.css`). FastAPI отдаёт её на `/web`, поэтому можно либо открыть `http://127.0.0.1:8000/web`, либо указать этот URL в настройках BotFather в качестве `Web App URL`.
- `src/` — черновики React/TS-компонентов из прошлого веб-UI. Они не участвуют в сборке и оставлены, если понадобится собрать полноценный SPA.

## Запуск
1) Поднимите backend: `cd backend && uvicorn app.main:app --reload`.
2) Откройте `http://127.0.0.1:8000/web` (или тот же путь, но уже из Telegram-клиента, если мини-апп опубликован).

## Функционал, который уже есть
- Регистрация/вход (JWT в `localStorage`, timezone сохраняется локально).
- CRUD задач с прогрессом (бинарный/процентный), фильтр по дате.
- CRUD привычек + отметки за день/неделю.
- CRUD напоминаний (time-based).
- Rule-based ассистент (`/assistant/message`), история диалога ведётся в UI.

## Что учтено под Telegram
- Поддержка `themeParams`, `HapticFeedback` и `WebApp.expand()`.
- Мобильная вёрстка, нижняя навигация, текстовые чипы вместо баннеров.
- Минимум внешних зависимостей: чистые HTML/CSS/JS без сборщика.
